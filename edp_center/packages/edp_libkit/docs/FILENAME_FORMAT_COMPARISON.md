# 文件名格式对比分析

## 两种方案对比

### 方案1：下划线分隔 `lib_config_1.00b.tcl`

**格式**：`lib_config_{version}.tcl`

**示例**：
- `lib_config.tcl`（最新版本）
- `lib_config_1.00b.tcl`
- `lib_config_2.00A.tcl`

**优点**：
- ✅ 下划线是常见的分隔符，广泛使用
- ✅ 在shell中处理简单，不需要特殊转义
- ✅ 版本号部分清晰独立
- ✅ 符合常见的命名约定（如 `file_v1.0.txt`）

**缺点**：
- ❌ 版本号本身用点分隔，用下划线连接可能不够统一
- ❌ 看起来可能不够自然

### 方案2：点分隔 `lib_config.1.00b.tcl`（推荐）

**格式**：`lib_config.{version}.tcl`

**示例**：
- `lib_config.tcl`（最新版本）
- `lib_config.1.00b.tcl`
- `lib_config.2.00A.tcl`

**优点**：
- ✅ **更统一**：版本号本身就用点分隔（`1.00b`），用点连接更自然
- ✅ **更直观**：`lib_config.1.00b.tcl` 看起来像版本号的扩展
- ✅ **符合语义**：点表示"属于"或"版本"，如 `package.json`, `config.1.0.tcl`
- ✅ **简洁美观**：整体格式更统一

**缺点**：
- ⚠️ 多个点可能在某些系统中被误解（但 `.tcl` 扩展名明确，不会有问题）
- ⚠️ 在极少数情况下可能需要转义（但通常不需要）

## 实际使用对比

### Shell脚本中使用

**下划线方案**：
```bash
source lib_config_1.00b.tcl  # 简单直接
```

**点方案**：
```bash
source lib_config.1.00b.tcl  # 同样简单，更统一
```

两者在shell中使用都很方便，没有明显区别。

### TCL脚本中使用

**下划线方案**：
```tcl
source lib_config_1.00b.tcl
```

**点方案**：
```tcl
source lib_config.1.00b.tcl
```

两者在TCL中使用都很方便，点方案可能更符合TCL的命名风格（TCL中常用点作为命名空间分隔符）。

## 我的建议

### 推荐：点分隔方案 `lib_config.1.00b.tcl`

**理由**：

1. **语义更清晰**
   - `lib_config.1.00b.tcl` 表示"lib_config的1.00b版本"
   - 点表示"属于"或"版本"，语义更明确

2. **格式更统一**
   - 版本号本身用点分隔（`1.00b`）
   - 文件名也用点连接，整体格式统一
   - 看起来更自然：`lib_config.1.00b.tcl` vs `lib_config_1.00b.tcl`

3. **符合常见约定**
   - 类似 `package.json`, `config.1.0.tcl` 等常见格式
   - 版本号作为"扩展"的概念更清晰

4. **实际使用无差异**
   - 在shell和TCL中使用都很方便
   - 不会造成任何实际问题

## 结论

虽然两种方案都可以，但我**推荐使用点分隔方案**（`lib_config.1.00b.tcl`），因为：
- 格式更统一自然
- 语义更清晰
- 符合常见约定
- 实际使用无差异

但如果团队更习惯下划线，使用下划线方案也完全可以。
