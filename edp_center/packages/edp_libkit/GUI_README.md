# EDP LibKit GUI - 图形用户界面

## 简介

轻量级的图形用户界面，让用户无需查看文档即可使用工具。

## 启动方式（无需安装！）

### 方式1：使用启动脚本（最简单，推荐）⭐

```bash
cd edp_center/packages/edp_libkit
python run_gui.py
```

**优点**：一键启动，无需设置任何路径

### 方式2：Python模块方式运行

```bash
cd edp_center
python -m packages.edp_libkit.gui
```

### 方式3：命令行启动（需要先安装）

```bash
# 如果已经安装
edp-libkit gui
```

**注意**：如果遇到 `command not found`，说明没有安装，使用方式1或2即可。

## 界面说明

### 主要功能区域

1. **Foundry选择** - 下拉框选择支持的Foundry（Samsung/SMIC/TSMC）
2. **工艺节点选择** - 下拉框选择节点（根据Foundry自动更新）
3. **库类型选择** - 单选按钮选择库类型（STD/IP/MEM）
4. **库路径** - 单个路径输入框 + 浏览按钮
5. **批量路径** - 多行文本框，支持：
   - 手动输入多个路径（每行一个）
   - 从文件读取路径列表
6. **输出目录** - 输出目录选择
7. **版本选择** - 三种模式：
   - 最新版本（默认）
   - 指定版本（输入版本号，如 1.00B）
   - 所有版本（生成所有版本的配置文件）
8. **执行日志** - 实时显示执行过程和结果

### 使用流程

1. **选择Foundry和节点**
   - 从下拉框选择Foundry
   - 节点列表会自动更新

2. **选择库类型**
   - 点击STD/IP/MEM单选按钮

3. **选择库路径**
   - **单个路径**：点击"浏览"按钮选择目录，或直接输入路径
   - **批量路径**：在多行文本框中输入多个路径（每行一个），或点击"从文件读取"

4. **选择输出目录**
   - 点击"浏览"按钮选择输出目录

5. **选择版本模式**（可选）
   - 默认使用最新版本
   - 如需指定版本，选择"指定版本"并输入版本号
   - 如需处理所有版本，选择"所有版本"

6. **生成配置**
   - 点击"生成配置"按钮
   - 在日志区域查看执行过程
   - 完成后会显示成功/失败统计

### 批量路径文件格式

批量路径文件是纯文本文件，每行一个路径：

```
/path/to/library1
/path/to/library2
/path/to/library3
```

支持注释（以 `#` 开头）：
```
# 这是注释
/path/to/library1
/path/to/library2
```

## 特点

- ✅ **无需文档**：界面直观，一看就会
- ✅ **实时反馈**：执行日志实时显示进度
- ✅ **批量处理**：支持一次处理多个库
- ✅ **错误提示**：友好的错误提示和验证
- ✅ **轻量级**：使用tkinter，无需额外依赖

## 注意事项

1. **路径优先级**：如果批量路径有内容，会优先使用批量路径；否则使用单个路径
2. **版本格式**：指定版本时，格式如 `1.00B`、`2.00A` 等
3. **输出目录**：会自动创建不存在的目录
4. **执行过程**：生成过程在后台线程执行，不会冻结界面

## 故障排除

### GUI无法启动

如果遇到 `ModuleNotFoundError: No module named 'tkinter'`：
- Windows/Linux: 通常已包含tkinter
- macOS: 可能需要安装：`brew install python-tk`

### 节点列表为空

- 检查Foundry是否正确选择
- 检查配置文件是否存在：`foundry_adapters/{foundry}/*.config.yaml`

### 生成失败

- 查看日志区域的错误信息
- 检查库路径是否存在
- 检查输出目录是否有写入权限
